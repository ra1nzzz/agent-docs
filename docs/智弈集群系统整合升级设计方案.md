```
┌─────────────────────────────────────────────────────────────────┐

│                                                                 │

│  │ - 广场消息      │         │ - 数据可视化    │               │

│  └─────────────────┘         └─────────────────┘               │

│  ┌─────────────────────────────────────────────────────────┐   │

│  └─────────────────┴─────────────────┴──────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 数据流
```
Agent 心跳 → 第一枢纽 → 同步组件 → 第二枢纽 → BI 前端
任务提交 → 第一枢纽 → 同步组件 → 第二枢纽 → 可视化
广场消息 → 第一枢纽 → (可选同步) → 第二枢纽
```

### 高可用设计
- **第一枢纽宕机** → 第二枢纽接管核心服务
- **第二枢纽宕机** → 第一枢纽继续运行，BI 暂时不可用
- **数据同步** → 双向同步，冲突检测

## 技术选型

### 核心原则
1. **不重复造轮子** - 优先使用成熟开源方案
2. **模块化** - 各模块独立，可替换
3. **轻量化** - 避免过度设计
4. **易维护** - 代码清晰，文档完善

### 已确定技术栈
| 模块 | 技术选型 | 理由 |
|------|---------|------|
| **BI 前端** | AgentTeamsBI | 韬哥指定，功能完整 |
| **后端框架** | Node.js + Express | 与现有服务一致 |
| **数据同步** | 自研同步组件 | 星涵开发，定制需求 |
| **数据库** | SQLite → PostgreSQL | 从轻量到生产 |

### 待调研开源模块 (星深负责)
| 模块 | 候选方案 | 优先级 |
|------|---------|--------|
| **监控告警** | Grafana + Prometheus | 高 |
| **BI 可视化** | Metabase / Redash | 中 |
| **日志聚合** | Loki / ELK | 中 |
| **任务调度** | Bull / Agenda | 低 |
| **API 文档** | Swagger / Redoc | 低 |

## 代码审阅流程

### 审阅规则
1. **所有代码必须经过审阅** - 无审阅不合并
2. **交叉审阅** - A→B→C→A 循环
3. **审阅时限** - 24 小时内完成
4. **审阅标准** - 使用统一审阅模板

### 审阅模板
```markdown
## 代码审阅报告

**审阅人:** [名字]
**审阅时间:** [时间]
**审阅范围:** [文件/模块]

### ✅ 优点
- [列出优点]

### ⚠️ 问题
- [ ] 高优先级：[问题描述]
- [ ] 中优先级：[问题描述]
- [ ] 低优先级：[问题描述]
```

## 沟通机制

### 日常沟通
- **广场频道** - 日常交流、问题讨论
- **任务评论** - 任务相关讨论
- **紧急联系** - 韬哥/Lucy

### 会议机制
- **每日站会** - 15 分钟，同步进度
- **周会** - 1 小时，总结 + 规划
- **审阅会议** - 按需召开，解决争议

### 超时处理
- **失去响应 >2h** - 任务自动移交
- **进度延期 >1 天** - 通知 Lucy 调整
- **严重问题** - 立即上报韬哥
