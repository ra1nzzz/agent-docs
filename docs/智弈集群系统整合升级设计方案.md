1. **不重复造轮子** - 优先使用成熟开源方案
2. **模块化** - 各模块独立，可替换
3. **轻量化** - 避免过度设计
4. **易维护** - 代码清晰，文档完善

### ⚠️ 待优化项

#### 1. 🔐 认证与授权（高风险）

**问题**：
- 工作频道 API 未认证，任何人访问 localhost:7891 可操作

- 任务操作（叫停/取消）缺少权限控制

**建议**：

- API 端点添加 RBAC，如 `POST /api/tasks/:id/stop` 需 `task:admin` scope

#### 2. 📊 数据存储演进路径

**现状**：
- AgentTeamsBI 使用 JSON 文件（`task_events.json`, `usage.json`）
- 智弈枢纽数据源为 `tasks_source.json`（JSON）
- 适合单实例，但并发写入有风险（虽已加文件锁）

**优化**（与 Phase2 API 评审建议一致）：
- **短期**: 保持 JSON，但使用原子操作（已实现 `atomic_json_write`）
- **中期**（1-2个月）: 迁移到 **PostgreSQL**
  - 理由：与 Redash 共享，减少运维成本
  - 迁移策略：保留 JSON 作为缓存层，逐步切换
- **数据层抽象**：在 `dashboard/server.py` 中引入 Repository 模式

```python
# 建议抽象
class TaskRepository:
    def get_tasks(self, filters=None): pass
    def update_task(self, task_id, updates): pass
    def add_event(self, event): pass
# 实现：JsonTaskRepository → PostgresTaskRepository
```

---

#### 3. 🔄 与 Phase2 API 的协同

**现状**：
- 工作频道直接读取 `DATA / tasks_source.json`
- 智弈枢纽 API（Phase2）将提供 `/api/tasks` 端点
- 存在数据源不一致风险

**建议集成方式**：
```python
# 配置可选数据源
DATA_SOURCE = os.getenv('DATA_SOURCE', 'local')  # 'local' | 'api'

if DATA_SOURCE == 'api':
    # 从 Phase2 API 拉取
    tasks = requests.get(f'{ZHIYI_HUB}/api/tasks').json()
else:
    # 本地 JSON 文件（兼容旧版）
    tasks = load_local_json()
```

**优先级**：Phase2 API 上线后，工作频道应优先使用 API 数据源。

#### 4. 📈 可观测性缺失

**问题**：AgentTeamsBI 自身无监控指标

**建议**：在 `server.py` 中暴露 Prometheus metrics

```python
from prom_client import Counter, Histogram

requests_total = Counter('bi_requests_total', 'Total requests', ['endpoint'])
request_duration = Histogram('bi_request_duration_seconds', 'Request duration')

@app.middleware
def measure(request, call_next):
    start = time.time()
    response = call_next(request)
    duration = time.time() - start
    request_duration.observe(duration)
    requests_total.labels(endpoint=request.path).inc()
    return response

@app.get('/metrics')
def metrics():
    return generate_latest()
```

**Dashboard**: 复用 Phase1 的 Grafana，添加 AgentTeamsBI 监控面板。

#### 5. 🛡️ 安全性加固
