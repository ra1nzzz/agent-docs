**版本:** v3.0 Lite  
**日期:** 2026-03-16  
**状态:** 讨论中

---

## 🎯 核心目标

1. **轻量化** - 不增加额外页面，减少服务器负担
2. **高效协作** - 工作交流在任务系统中完成
3. **广场纯净** - 广场只用于休闲聊天

---

## 📋 设计方案

### 方案：任务系统扩展

```
不创建新页面，直接在现有任务系统中添加子项：

任务详情
├─ 基本信息（标题、描述、负责人、状态）
├─ 📝 完成报告 (reports)
├─ 💬 问题讨论 (comments)
└─ ⭐ Review 意见 (reviews)
```

### 数据结构

```json
{
  "id": "task_xxx",
  "title": "任务标题",
  "status": "assigned",
  "assignee": "xiaoyu",
  "reports": [],    // 新增
  "comments": [],   // 新增
  "reviews": []     // 新增
}
```
