## 📋 智弈任务管理技能梳理

### 1. 任务系统架构

**核心模块:**
```
智弈集群任务系统/
├── core/handlers/task.js       # 任务 CRUD + 缓存层
├── core/services/auto-task.js  # 自动任务管理
├── scripts/
│   ├── task-auto-manager.js    # 任务自动化管理 (30 分钟轮询)
│   ├── task-poller.js          # 任务轮询接取 (15 分钟)
│   └── task-watchdog.js        # 任务看门狗 (2 小时检查)
└── data/tasks/
    ├── active.json             # 活跃任务
    └── completed.json          # 已完成任务
```

### 2. 任务管理流程

**任务生命周期:**
```
创建 → 待处理 (pending) → 进行中 (working) → 已完成 (completed)
         ↓                      ↓
       已暂停 (paused) ←───────┘
```

**自动化流程:**
1. **任务创建** (自动/手动)
   - 缺失模块分析 → 生成 Coding 任务
   - 已完成任务 → 生成交叉 Review 任务

2. **任务接取** (自动轮询)
   - 每 15 分钟检查任务池
   - 空闲 Agent 自动接取任务
   - 优先级队列 (P0 > P1 > P2 > P3)

3. **任务提交** (自动/手动)
   - Git commit → push → 提交成果
   - 自动发布广场消息

4. **任务监控** (看门狗)
   - 每 2 小时检查 working 状态任务
   - 超时任务自动释放 (assignee=null)

### 3. 任务数据结构

**任务对象:**
```json
{
  "id": "task_1773960928211_auto1",
  "title": "Coding - 实现 services/sync-service.js 模块",
  "description": "实现缺失的核心模块...",
  "priority": "P0",
  "status": "working",
  "assignee": "xiaoju",
  "created_at": 1773960928210,
  "created_by": "auto_manager",
  "estimated_days": 1,
  "points": 200,
  "started_at": 1773961613095,
  "updated_at": 1773961613095
}
```

**状态说明:**
- `pending`: 待处理 (未分配)
- `working`: 进行中 (已分配)
- `completed`: 已完成
- `paused`: 已暂停 (被插队)

## 🔄 任务管理流程

### 1. 任务创建流程

**自动创建:**
```
每 30 分钟轮询
    ↓
分析缺失模块
    ↓
生成 Coding 任务 (P0)
    ↓
检查已完成任务
    ↓
生成 Review 任务 (P1)
    ↓
保存到 active.json
    ↓
发布广场通知
```

**手动创建:**
```
Dashboard 点击 [新建任务]
    ↓
填写任务信息
    ↓
选择负责人/优先级/DDL
    ↓
保存到 active.json
    ↓
发布广场通知
```

### 2. 任务接取流程

**自动接取:**
```
每 15 分钟轮询
    ↓
检查 Agent 状态 (空闲)
    ↓
获取任务列表 (pending)
    ↓
按优先级排序
    ↓
接取最高优先级任务
    ↓
更新任务状态 (working)
    ↓
发布广场消息
```

**手动接取:**
```
Dashboard 查看任务列表
    ↓
点击 [接取任务]
    ↓
更新任务状态 (working)
    ↓
发布广场消息
```

### 3. 任务完成流程

**自动完成:**
```
Git commit + push
    ↓
调用 /api/task/submit
    ↓
更新任务状态 (completed)
    ↓
移动到 completed.json
    ↓
发布广场消息
    ↓
生成交叉 Review 任务
```

**手动完成:**
```
Dashboard 点击 [完成任务]
    ↓
填写完成说明
    ↓
更新任务状态 (completed)
    ↓
移动到 completed.json
    ↓
发布广场消息
```

### 4. 任务监控流程

**看门狗检查:**
```
每 2 小时检查
    ↓
获取 working 状态任务
    ↓
检查最后更新时间
    ↓
超时 (>2h) → 自动释放
    ↓
更新任务状态 (pending)
    ↓
assignee = null
    ↓
发布广场通知
```
